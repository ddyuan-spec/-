一、申请限制

原型链接

客户端原型
1.1 需求背景

当前系统对售后申请次数没有限制，可能导致恶意退款。用户申请售后时需要校验申请次数，超过限制后不允许申请。

1.2 需求目标
  • 订单维度：同一订单累计申请售后次数 ≤ 3次
  • 商品维度：同一订单同一商品累计申请售后次数 ≤ 3次
  • 超过限制后，提示用户不可再申请
1.3 功能内容
字段名称功能说明业务规则
订单维度限制同一订单累计申请售后次数限制同一订单累计申请售后次数 ≤ 3次；超过3次后不允许发起新的售后申请
商品维度限制同一订单同一商品累计申请售后次数限制同一订单同一商品累计申请售后次数 ≤ 3次；超过3次后不允许对该商品发起售后申请
次数统计统计用户已申请的售后次数统计维度：用户ID + 订单ID + 商品SKU；统计范围：所有售后单（含已完成、已拒绝、已关闭）
限制提示超过限制时的提示文案显示："该订单/商品已超过最大售后申请次数（3次），无法再申请售后"
申请入口超过限制时的申请入口状态超过限制时，申请入口按钮置灰或隐藏，不允许点击
1.4 影响端口
端口功能范围
客户端售后申请入口校验
系统售后申请次数统计与校验
二、部分退款（含组合支付）

原型链接

商家端原型
2.1 需求背景

用户可能需要对订单中的部分商品申请退款。当前系统支持整单退款，不支持部分退款。需要支持按商品维度进行部分退款，并处理组合支付情况下的退款金额分配。

2.2 需求目标
  • 支持按商品维度选择退款商品
  • 支持输入退款金额
  • 支持组合支付（现金+余额）的退款金额分配
  • 退款金额不能超过商品实付金额
2.3 功能内容
2.3.1 退款商品选择
字段名称功能说明业务规则
商品列表展示订单中可退款的商品展示订单所有商品，支持多选；已退款的商品显示已退数量，不可再次退款
商品选择用户选择要退款的商品勾选框选择；可选择单个或多个商品；已退完的商品不可选
退款数量选择每个商品的退款数量默认1；可手动输入；不超过该商品购买数量-已退款数量
已退标识显示商品已退款信息已退款商品显示"已退款X件"；不可再选
2.3.2 退款金额计算
字段名称功能说明业务规则
商品实付金额该商品的实际支付金额商品支付金额 = 商品总价 - 使用的优惠券分摊 - 积分抵扣分摊
退款上限可退金额上限不能超过商品实付金额；不能超过该商品购买数量 × 单价
退款金额输入用户输入退款金额手动输入；范围：0.01元 ~ 退款上限；保留2位小数
退款金额校验校验输入金额是否合法不能为空；不能为0；不能超过退款上限；不能低于0.01元
2.3.3 组合支付退款分配
字段名称功能说明业务规则
支付方式订单的支付方式支持：仅现金、仅余额、现金+余额组合
退款分配逻辑多支付方式时的退款分配优先退余额，余额不足则退现金；分配比例 = 支付比例
退款计算公式计算各支付方式的退款金额余额退款 = 退款金额 × (余额支付金额 ÷ 实付总金额)；现金退款 = 退款金额 - 余额退款
最小退款单位退款金额最小单位以分为单位，0.01元；各渠道退款金额四舍五入
2.3.4 退款原因
字段名称功能说明业务规则
退款原因用户选择退款原因下拉选择：商品损坏/错漏发/商品与描述不符/其他
退款说明用户填写退款说明可选填；最多200字
图片上传用户上传凭证图片可选；最多上传3张；支持jpg、png格式
2.4 影响端口
端口功能范围
客户端部分退款申请入口、退款商品选择、退款金额输入
商家端退款审核、退款金额确认
系统退款金额计算、退款分配、资金退回
三、商家退款展示

原型链接

商家端原型
3.1 需求背景

商家端订单详情页缺少退款信息展示，运营无法查看退款明细。需要增加退款信息模块，展示退款单号、退款金额、退款方式等详细信息。

3.2 需求目标
  • 商家端订单详情页展示退款信息模块
  • 展示退款单号、退款金额、退款方式、退款时间等
  • 支持组合支付的退款明细展示
3.3 功能内容
字段名称功能说明业务规则
退款单号退款记录的唯一标识格式：TK + 年月日 + 序号
退款金额本次退款的金额显示退款总金额，单位：元
退款方式退款原路返回的支付方式显示：余额退回、现金退回（微信/支付宝/银行卡）
退款时间退款操作的时间格式：YYYY-MM-DD HH:mm:ss
退款状态退款的状态退款中、已退款、退款失败
退款类型退款的类型整单退款、部分退款
3.3.1 组合支付退款明细
字段名称功能说明业务规则
余额退款余额支付的退款金额显示余额退款金额；若未使用余额支付则显示"-";
现金退款现金支付的退款金额显示现金退款金额；若未使用现金支付则显示"-";
退款明细展示退款明细列表一条退款记录可能包含多个退款明细（余额+现金）
3.4 影响端口
端口功能范围
商家端订单详情页退款信息展示
系统退款记录生成、退款状态更新
四、客户端退款详情页

原型链接

客户端原型
4.1 需求背景

客户端缺少退款详情页，用户无法查看退款进度、退款金额、退款方式等详细信息。需要新增退款详情页，提供完整的退款信息展示。

4.2 需求目标
  • 新增客户端退款详情页
  • 展示退款状态、退款金额、退款方式、退款进度
  • 支持退款中、退款成功、退款失败三种状态
4.3 功能内容
4.3.1 退款状态卡片
字段名称功能说明业务规则
状态图标展示退款状态的图标退款中：时钟图标；退款成功：勾选图标；退款失败：叉号图标
状态标签展示退款状态文字退款中/退款成功/退款失败
状态描述展示退款状态的补充说明退款中："预计1-3个工作日到账"；退款成功："YYYY-MM-DD HH:mm到账"；退款失败："退款失败，已恢复订单"
4.3.2 退款信息展示
字段名称功能说明业务规则
退款单号退款记录的唯一标识格式：TK + 年月日 + 序号
退款金额本次退款的金额大字号展示；单位：元
退款方式退款原路返回的支付方式显示：退回原支付方式/退回余额
申请时间用户提交退款申请的时间格式：YYYY-MM-DD HH:mm:ss
退款原因用户申请的退款原因显示用户选择的退款原因
4.3.3 退款进度
字段名称功能说明业务规则
时间轴展示退款进度时间轴按时间倒序展示；已完成节点显示绿色；当前进行中节点高亮
进度节点每个进度节点的信息包含：时间戳、状态、操作说明
失败原因退款失败时的原因说明仅退款失败状态显示；红色高亮展示
4.3.4 页面操作
字段名称功能说明业务规则
完成按钮退款成功后的确认按钮文案："完成"；点击返回上一页
查看订单退款失败后的跳转按钮文案："查看订单"；点击跳转至订单详情页
联系客服用户联系客服的入口文案："如有疑问，联系客服"；点击跳转至客服页面
4.4 影响端口
端口功能范围
客户端退款详情页展示、退款进度
系统退款状态同步、退款详情查询
