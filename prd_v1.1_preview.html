一、部分退款功能

原型链接

商家端原型 客户端原型
1.1 需求背景

用户可能需要对订单中的部分商品申请退款。当前系统支持整单退款，不支持部分退款。需要支持按商品维度进行部分退款，并处理组合支付情况下的退款金额分配。同时客户端缺少退款详情页，用户无法查看退款进度和详情。

1.2 需求目标
  • 商家端：支持按商品维度选择退款商品，输入退款金额，处理组合支付退款
  • 客户端：新增退款详情页，展示退款状态、金额、进度等信息
  • 退款金额不能超过商品实付金额
1.3 商家端功能内容
1.3.1 售后工单详情页

本页面包含两个主要场景：【待确认退款场景】商家确认收货前，可操作退款或拒绝；【已退款场景】退款完成后，展示退款记录，操作按钮置灰。

字段名称功能说明数据来源业务规则
订单状态展示当前售后单状态售后单表.aftersale_status【待确认退款场景】：显示"退货退款 待商家确认收货（退货未入库）"；【已退款场景】：显示"退货退款 已退款（部分退款/全部退款）"
状态说明状态补充说明固定文案（根据状态动态匹配）【待确认退款场景】："如您逾期未响应申请，视作同意买家申请，系统会自动退款给买家"；【已退款场景】："售后单已完成，订单状态已更新"
超时倒计时剩余处理时间系统自动计算（售后单表.expire_time - 当前时间）【待确认退款场景】显示：剩余X天X时X分X秒；【已退款场景】不显示
确认收货并同意退款按钮商家确认收货并退款-【待确认退款场景】可点击，弹出退款确认弹窗；【已退款场景】置灰不可点击
拒绝退货退款按钮商家拒绝退款申请-【待确认退款场景】可点击，进入拒绝流程；【已退款场景】置灰不可点击
1.3.4 退款记录（退款后展示）

退款记录以表格形式展示，位于协商历史上方。仅退款成功后展示，可能存在多条退款记录。底部汇总展示订单支付信息和累计退款情况。

字段名称功能说明数据来源业务规则
退款规则提示顶部提示退款分配规则固定文案显示："退款规则：优先退还储值支付部分，储值不足时退还在线支付部分"
退款时间执行退款操作的时间系统自动生成（退款记录表.refund_time）格式：YYYY-MM-DD HH:mm:ss；按时间倒序排列（最新在前）

退款类型本次退款的类型商家在退款弹窗中选择（退款记录表.refund_type）枚举值：部分退款（蓝色标签）/ 全部退款（橙色标签）
退款金额本次退款的金额商家在退款弹窗中输入（退款记录表.refund_amount）精确到小数点后两位；单位：¥；红色加粗显示
储值退款本次从储值支付中退还的金额系统自动计算（退款记录表.store_refund_amount）计算规则：优先退储值，退款金额 ≤ 剩余储值可退时，储值退款 = 退款金额，在线支付退款 = 0；退款金额 > 剩余储值可退时，储值退款 = 剩余储值可退，在线支付退款 = 退款金额 - 储值退款。精确到小数点后两位
在线支付退款本次从在线支付中退还的金额系统自动计算（退款记录表.online_refund_amount）计算规则：退款金额 - 储值退款。精确到小数点后两位
退款状态退款处理的状态系统根据退款处理结果自动更新（退款记录表.refund_status）枚举值：退款中（橙色）/ 已退款（绿色）/ 退款失败（红色）
订单金额（汇总行）订单原始金额订单表.order_amount精确到小数点后两位；单位：¥
订单储值支付（汇总行）订单中储值支付的金额订单表.store_pay_amount精确到小数点后两位；单位：¥
订单在线支付（汇总行）订单中在线支付的金额订单表.online_pay_amount精确到小数点后两位；单位：¥
累计已退款（汇总行）该售后单累计退款金额系统统计当前售后单所有退款记录的退款金额之和红色显示；精确到小数点后两位；单位：¥
剩余可退（汇总行）当前还可退款的金额系统计算：订单金额 - 累计已退款橙色显示；精确到小数点后两位；单位：¥；剩余可退为0时不可再发起退款

2.2 需求目标

  • 展示退款单号、退款金额、退款方式、退款时间等

三、仅换货功能

原型链接

商家端原型
3.1 需求背景

当前售后类型包括退货退款、仅退款，缺少"仅换货"类型。用户收到商品后，商品存在质量问题或规格不符，但用户仍希望保留该商品（更换同款新品），此时需要支持仅换货流程。

3.2 需求内容

3.2.1 新增售后类型：仅换货

字段名称功能说明业务规则
售后类型用户申请售后时选择新增"仅换货"选项，与"退货退款"、"仅退款"并列
适用场景商品质量问题/规格不符，用户希望更换同款仅支持同款商品换货（相同SKU），不支持换其他商品
订单状态要求已发货或已收货订单可申请待发货订单不支持仅换货（未发货可直接修改订单）

3.2.2 仅换货流程

流程节点用户端商家端系统
1. 用户申请选择"仅换货"，填写原因，提交申请-生成售后单，状态：待商家审核
2. 商家审核-审核换货申请，同意/拒绝同意→状态变为"待用户退货"；拒绝→售后单关闭
3. 用户退货填写退货物流单号，寄回商品-状态变为"待商家确认收货"
4. 商家收货-确认收货并验货，生成换货订单自动生成新订单（换货订单），原订单保持已完成
5. 换货发货查看换货订单物流信息对换货订单进行发货操作换货订单走正常订单发货流程

3.2.3 换货订单规则

字段名称功能说明业务规则
换货订单生成商家确认收货后自动生成新订单与原订单关联（记录换货关系）；订单金额：¥0（免支付）；商品信息：与原商品相同SKU
换货订单状态走正常订单状态流转待发货→待收货→已完成
换货订单售后换货订单是否支持再次售后支持，走正常售后流程（有金额限制，见3.2.4）
收益计算换货订单是否产生收益换货订单¥0支付，不产生收益

3.2.4 仅换货售后限制

限制类型规则说明
次数限制一个订单仅支持一次仅换货申请；换货后的新订单可再次申请售后（包括退货退款、仅退款、仅换货）
金额限制换货订单金额为¥0，不支持退款类售后（退货退款、仅退款）；换货订单仅支持再次换货
商品限制仅支持同款换货（相同SKU）；不支持跨SKU换货
3.3 端口范围
端口功能范围
用户端售后申请页面新增"仅换货"选项；售后详情展示换货进度；查看换货订单
商家端售后审核支持仅换货类型；确认收货生成换货订单；换货订单管理
系统售后单类型扩展；换货订单自动生成逻辑；订单关联关系记录
3.4 状态流转
售后单状态触发条件说明
待商家审核用户提交仅换货申请商家需在48小时内处理
待用户退货商家审核通过用户需填写退货物流
待商家确认收货用户填写退货物流商家验货后确认收货
换货处理中商家确认收货系统生成换货订单，商家发货
已完成换货订单完成用户确认收货或自动完成
已关闭商家拒绝或用户取消售后单关闭
