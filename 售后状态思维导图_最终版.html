🔒 已关闭状态（并行终态）
可从任意订单状态进入已关闭：
• 待付款 → 用户取消 / 支付超时 → 已关闭
• 待发货 → 仅退款（全部） → 已关闭
• 待收货 → 仅退款/退货退款（全部） → 已关闭
• 已完成 → 退货退款（全部） → 已关闭
已关闭为终态，不可申请任何售后，不计算收益
订单售后系统
💰 待付款
👤 用户操作
  • 用户 支付订单
  • → 进入待发货
👤 客户取消
  • 用户 取消订单
  • 超时 支付超时自动关闭
  • → 订单关闭
📦 待发货
（仅退款）
📋 备货中
📌 进入条件：
🏪 本地发货：导出订单号 → 备货中
☁️ 云仓发货：调用接口 → 备货中
📌 发货路径：
🏪 本地发货：上传物流单号 → 待收货
☁️ 云仓发货：返回物流单号 → 待收货
📝 说明：
后台状态，用户端仍显示"待发货"，无特殊业务逻辑
💰 仅退款
  • 用户 申请仅退款
  • 运营 拦截物流 → 审核
  • 运营 审核通过 → 退款
💵 退款结果：
→ 退款成功 → 订单已关闭
→ 驳回 → 订单恢复待发货
👤 商家发货
  • 运营 填写物流单号
  • → 进入待收货
🚚 待收货
⏰ 系统自动收货
  • 系统 发货后7天无物流签收
  • 系统 自动确认收货
  • → 订单进入已完成
⚠️ 存在售后单
  • 用户 申请售后 → 售后单生效
  • 系统 自动收货暂停
  • 系统 自动收货计时器暂停
  • → 订单卡在待收货
  • → 售后完成后恢复计时
💰 仅退款
  • 用户 申请仅退款
  • 运营 拦截物流
  • 运营 根据拦截结果审核
⚡ 拦截结果决定审核
  • ✓ 拦截成功 → 审核通过 → 退款 → 订单已关闭
  • ✕ 拦截失败 → 驳回 → 恢复待收货
📦 退货退款
  • 用户 申请退货退款
  • 运营 审核通过
  • 用户 填写物流单号并寄回
  • 运营 确认收货 → 退款
💵 退款结果：
→ 全部退款 → 订单已关闭
→ 部分退款 → 订单已完成（计算收益）
🔄 仅换货
  • 用户 申请仅换货
  • 运营 审核通过
  • 用户 填写物流单号并寄回
  • 运营 确认收货 → 重新发货
  • → 换货完成，订单仍为待收货
✅ 已完成
📦 退货退款
  • 用户 申请退货退款
  • 运营 审核通过
  • 用户 填写物流单号并寄回
  • 运营 确认收货 → 退款
💵 退款结果：
→ 全部退款 → 订单已关闭
→ 部分退款 → 订单保持已完成
🔄 仅换货
  • 用户 申请仅换货
  • 运营 审核通过
  • 用户 填写物流单号并寄回
  • 运营 确认收货 → 重新发货
  • → 换货完成，订单仍为已完成
🔒 已关闭
🚫 终态说明
  • 订单已关闭，不可申请任何售后
  • 全部退款/全部退货退款后进入此状态
  • 用户取消订单 / 支付超时进入此状态
  • 不计算收益
订单状态 售后类型 退款金额 订单最终状态
待付款 — — 待发货 / 已关闭
待发货 仅退款 全部退款 已关闭
（驳回） 恢复待发货
待收货 仅退款 全部退款 已关闭
退货退款 全部退款 已关闭
部分退款 已完成（计算收益）
已完成 退货退款/仅换货 全部退款 已关闭
部分退款 保持已完成
已关闭 — — 终态，不可申请售后
操作角色
用户 客户操作
运营 商家/运营
系统 系统自动
状态标签
已关闭 全部退款后订单关闭
已完成（计算收益） 部分退款后订单完成，计算收益
保持已完成 部分退款，保持订单原状态
